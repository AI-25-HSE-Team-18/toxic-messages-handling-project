# T-Guard

## Тема проекта

Тема проекта: 15. Реализация веб-сервиса определения токсичности текстовых сообщений.

## Описание проекта

Данный проект представляет собой веб-сервис, использующий методы машинного обучения и глубокого обучения для определения токсичности текстовых сообщений.

## Цели проекта

- Разработать production-ready веб-сервис для автоматического определения токсичности текстовых сообщений
- Создать масштабируемое решение, способное обрабатывать большие объемы текстовых данных в режиме реального времени
- Создать систему метрик и бенчмарков для оценки качества детекции токсичности
- Исследовать влияние различных методов предобработки текста и feature engineering на качество классификации
- Выяснить, в каких моментах классические ML-модели могут быть релевантны, в каких внедрение DL-моделей является необходимым для достижения целей проекта по качеству

## Результаты экспериментов

### Классический ML

| Модель | Macro F1 |
|--------|----------|
| Linear SVM | ~0.91 |
| CatBoost | ~0.878 |

### Deep Learning

| Модель | Macro F1 |
|--------|----------|
| GRU (1 layer, cw) | 0.815 |
| BiLSTM (2 layers, cw) | 0.815 |
| GRU (2 layers, cw) | 0.810 |
| BiGRU (1 layer, cw) | 0.810 |

### BERT

Fine-tuning `DeepPavlov/rubert-base-cased` (`finetune_BERT.ipynb`): accuracy до ~0.90 на бинарной классификации.

### Выводы

- Дотюненный классический ML (Linear SVM) достигает macro F1 ~0.91 - выше, чем у DL-моделей на тех же данных.
- Нелинейный ML (CatBoost) - до ~0.878; сложность модели не гарантирует прирост качества.
- DL (GRU, BERT) конкурентен по ROC-AUC, но проигрывает по macro F1 и требует больше ресурсов на инференс.

## Датасеты

Данные собраны из открытых русскоязычных источников и приведены к единой схеме:

| Поле | Описание |
|------|----------|
| `raw_text_id` | Идентификатор записи |
| `dataset_id` | Идентификатор исходного датасета |
| `source_platform` | Платформа-источник (форум, сайт и т.д.) |
| `text_raw` | Исходный текст сообщения |
| `is_toxic` | Бинарная метка (0 — нетоксично, 1 - токсично) |
| `toxicity_type` | Тип токсичности (мультиклассовая разметка) |

Источники (Kaggle):

- [blackmoon/russian-language-toxic-comments](https://www.kaggle.com/datasets/blackmoon/russian-language-toxic-comments)
- [alexandersemiletov/toxic-russian-comments](https://www.kaggle.com/datasets/alexandersemiletov/toxic-russian-comments)
- [nigula/russianinappropriatemessages](https://www.kaggle.com/datasets/nigula/russianinappropriatemessages)

## API

### Публичные endpoints
- `GET /` - Информация об API
- `POST /register` - Регистрация нового пользователя (роль USER)

### Пользовательские endpoints
- `GET /history` - История запросов пользователя
- `POST /forward` - Анализ токсичности текста

### Админские endpoints
- `GET /users` - Список всех пользователей
- `GET /history/stats` - Статистика по всем запросам
- `DELETE /history` - Очистка истории запросов
